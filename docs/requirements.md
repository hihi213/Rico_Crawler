# 요구사항 (누리장터 입찰공고 크롤러)

## 범위
- 대상: 누리장터 입찰공고 목록/상세 페이지
- 상세 페이지에서 가능한 모든 필드 수집
- 동적 렌더링 대응(브라우저 자동화 필요)

## 기능 요구사항
- 목록 페이지: 페이지네이션 처리 및 상세 링크/식별자 수집
- 상세 페이지: 핵심 필드 추출, 라벨 누락/변형 허용
- 재개: 페이지마다 체크포인트 저장, 재실행 시 이어서 수집
- 재시도: 일시적 오류는 재시도, 영구 오류는 로그 후 스킵
- 중복 방지: DB UNIQUE 제약 필수, 필요 시 in-memory set
- 출력: 정규화된 데이터를 DB 저장 + 샘플 JSON/CSV 산출

## 비기능 요구사항
- 설정: 모든 URL/타임아웃/간격/수집 범위는 `config.yaml`에서 관리
- 로깅: 구조화 로그(가능하면 structlog), 페이지/식별자 컨텍스트 포함
- 안정성: 아이템 단위 실패는 전체 중단 없이 계속 진행
- 재현성: 새 환경에서 5분 내 실행 가능(크롤링 시간 제외)
- 테스트: pytest 커버리지 목표 명시, 핵심 로직 단위 테스트
- 문서: README에 실행 방법/가정/제한/샘플 포함

## 수용 기준
- 수동 개입 없이 N 페이지 크롤링 가능
- Ctrl+C 후 재실행 시 마지막 체크포인트부터 재개
- DB UNIQUE 제약으로 중복 저장 차단
- 날짜/금액/식별자 정규화 후 저장
- 로그에서 페이지/식별자로 추적 가능
